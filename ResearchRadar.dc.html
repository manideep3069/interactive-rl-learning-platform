How this stays honest

Two mechanisms. Live: your browser queries arXiv's public API on every visit, so the paper list is minutes old. Weekly: a GitHub Action sweeps arXiv every Monday, curates the highest-signal RL papers into a committed snapshot, and appends a changelog entry — that's the trail above. Chapter tags come from a transparent keyword heuristic (imperfect on purpose — better a visible rule than an invisible guess). Lesson text is never rewritten by a robot: the sweep proposes, the changelog discloses, and a human merges. Sources beyond arXiv (ICLR/NeurIPS via OpenReview, journal feeds) plug into the same pipeline.
