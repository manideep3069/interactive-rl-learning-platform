Chapter 8 · Model-Based RL & Planning

Learn the world, then plan inside it

Chapter 2 planned with a known model and then we threw the model away for six chapters. This chapter brings it back — first as a search tree over a perfect simulator (MCTS, AlphaZero), then as a learned model (MuZero), then as a learned model an agent dreams inside (Dreamer). This is the sample-efficiency end of RL, and increasingly the robotics end too.

Part 1

MCTS — grow the tree where it matters

Given a simulator you can reset, planning is search. Monte Carlo Tree Search repeats four steps: select a path down the tree, expand a leaf, evaluate it (a rollout, or later a value network), and back up the result. The intelligence is all in the selection rule — at every node, pick the child maximizing:

Average result so far, plus a bonus that shrinks as a move gets visited. Rarely-tried moves stay tempting; bad moves get tried enough to be condemned, then ignored. The tree grows asymmetrically — deep where the game is sharp, shallow where it's decided. Hold that formula: it is a bandit rule (Chapter 9's UCB) applied at every node of a search.

Analogy

Reading ten chess continuations: you don't read each to the same depth. Forcing lines get twenty moves of attention; obviously losing lines get two. MCTS is that instinct, made mechanical and provably calibrated.

In the wild

MCTS ran Go engines for a decade before deep learning arrived, and it never left: chip placement, compiler optimization, theorem proving, and molecule design all reuse the same four-step loop wherever a cheap simulator exists. Build one yourself — Project 6.

Part 2

AlphaZero and MuZero — search as teacher

AlphaZero closes a beautiful loop. A network gives each position a policy prior and value ; MCTS, guided by them, thinks for ~800 simulations and produces better move probabilities than the raw network. Then the network is trained to predict what search concluded:

Search improves the policy; the policy improves the search. That is policy iteration — Chapter 2's engine — with MCTS as the improvement operator and self-play as the environment. From random weights to superhuman chess in hours, no human games needed.

MuZero drops the last crutch: the rules. It learns a model to plan with — an encoder , a dynamics network , a predictor — and crucially, the learned state is never asked to reconstruct the board. It only has to predict what planning needs: policy, value, reward. A model of what matters, not of what's there. Same search, same loss, but now it plays Go and Atari — no rulebook anywhere.

The value-equivalence idea

Classic model-based RL learned pixel-predictors that wasted capacity on clouds and shadows, then compounded small errors into hallucinations. MuZero's answer: judge the model only by the quality of the plans it supports. If two models yield the same values, they are the same model for RL purposes. This one idea quietly reorganized the field.

Part 3

Dreamer — learn inside the dream

MCTS plans at decision time; Dreamer plans at training time. It learns a recurrent latent world model from replayed experience, then trains an actor-critic — Chapter 5's machinery, unchanged in spirit — entirely on imagined rollouts inside the latent space. Real steps train the dream; dreamed steps train the policy:

# the Dreamer loop, in caricature
for step in real_world:
    buffer.add(env.step(actor(obs)))          # act, remember
    world_model.train(buffer.sample())        # dream better
    z = world_model.encode(buffer.sample())
    traj = world_model.imagine(actor, z, horizon=15)   # 15 steps, no env
    actor_critic.train(traj)                  # learn entirely in latent space

Every real step is reused thousands of times as fuel for imagination — which is why DreamerV3 learns Atari tasks from ~200k frames where model-free methods want 50M+, and why it was the first algorithm to mine diamonds in Minecraft from scratch. One fixed set of hyperparameters, 150+ tasks. The catch, as always: the policy is only as good as the dream is honest.

Analogy

A pilot who flies once a week but rehearses daily in a flight simulator built from their own logbook. Cheap practice, honest only where the logbook was. The better the simulator, the fewer real flights mastery costs.

In the wild — 2026

World models are the current frontier: DeepMind's Genie line learns interactive worlds from video, robotics labs fine-tune foundation world models per task, and the emerging recipe — pretrain a world model, then train policies inside it — mirrors what LLMs did to NLP. Watch this space on the radar.
